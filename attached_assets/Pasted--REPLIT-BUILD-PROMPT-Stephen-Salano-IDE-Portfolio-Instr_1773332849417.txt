# REPLIT BUILD PROMPT — Stephen Salano IDE Portfolio
## Instructions for use: Copy everything below this line into Replit Agent exactly as written. Do not summarise it. Do not paraphrase it. Paste the full text.

---

Build a personal portfolio website for Stephen Salano. The portfolio is designed to look and feel like an open IntelliJ IDEA project in the JetBrains New UI style. The visitor experiences the portfolio as if they opened an IDE and found a developer's project already running. Every design and layout decision below is final and must not be changed, approximated, or creatively interpreted. If something is not specified, use the closest neutral option — do not invent.

---

## TECH STACK

Build this as a single-page React application using Vite. Use React Router for navigation between the five pages. Use plain CSS modules or a single global CSS file — do not use Tailwind CSS, styled-components, or any other CSS framework. Load JetBrains Mono and Inter from Google Fonts. Use Lucide React for all icons. No other external UI libraries.

---

## COLOUR TOKENS

Define these as CSS custom properties on `:root` and use them everywhere. Never use a raw hex value outside of this declaration block.

```css
:root {
  --bg-deepest: #1E1F22;

  --bg-tab-active: #4E5157;
```

## TYPOGRAPHY

Load from Google Fonts:
- JetBrains Mono — weights 400 and 500
- Inter — weights 400, 500, and 600

JetBrains Mono is used for: the file tree labels, tab bar filenames, status bar text, breadcrumb, the logo `<salano />`, all `//` and `/* */` comment labels in the content area, tech pill text, and the `.md` extension colouring in the tree.

Inter is used for: all body copy, paragraph text, experience bullets, project descriptions, section headings inside the content area, form labels, and CTA button text.

No other fonts. These two fonts are the entire typographic palette.

## OVERALL LAYOUT STRUCTURE

The layout is a full-viewport IDE frame. Nothing outside the status bar scrolls — only the content area scrolls internally. The layout is composed of five zones:

1. Activity Bar — 40px wide, full height minus status bar, far left
2. File Tree Panel — 220px wide, full height minus status bar, next to activity bar
3. Tab Bar — 36px tall, sits above the content area to the right of the file tree
4. Breadcrumb Bar — 28px tall, sits directly below the tab bar
5. Content Area — fills remaining width and height, only scrollable zone
6. Status Bar — 22px tall, full viewport width, pinned to bottom

The entire layout must use `height: 100vh` and `overflow: hidden` on the root. Only the content area has `overflow-y: auto`.

## ZONE 3 — TAB BAR

Height: 36px. Background: var(--bg-panel). Bottom border: 1px solid var(--border-subtle). Sits to the right of the file tree, above the content area.

On first load, only "Home.md" tab is open. As the visitor navigates to other pages, new tabs are added and persist — they do not auto-close. The tab bar therefore starts with one tab and can accumulate up to five.

Each tab is a flex row: 12px left padding, 8px right padding, filename in 12px JetBrains Mono. Max width per tab: 160px.

Inactive tab: var(--text-secondary) text, transparent background, 1px right border var(--border-subtle).

Active tab: var(--text-primary) text, background var(--bg-tab-active), and critically a 2px solid var(--green) border on the TOP edge of the tab only. This green top border is the most important detail in the entire tab bar — do not omit it or move it to a different edge.

Each tab has a small X icon (Lucide X, 10px) on the right side. The X is invisible at rest (opacity: 0) and fades in (opacity: 1) when the tab is hovered. Clicking the X closes the tab and navigates to the previous open tab, or to Home.md if all others are closed. Home.md cannot be closed — its X icon should not be shown at all.

New tab entrance animation: slide in from the right, width animates from 0 to full over 200ms with ease-out.
